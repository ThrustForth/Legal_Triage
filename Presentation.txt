Legal Triage System - Progress Presentation
Executive Summary

Built a production-ready AI-powered legal triage system for legal aid organizations that combines eligibility screening, process guides, and semantic search over indexed legal documents using RAG (Retrieval-Augmented Generation) architecture.
Features & Implementation
1. Eligibility Screening System

What it does:

    Screens clients for legal aid eligibility based on income, household size, legal issue, and geography

    Assigns priority levels (1=Critical, 2=High, 3=Normal)

    Shows criteria breakdown for ineligible clients

    Provides referral information for clients who don't qualify

How it was implemented:

    Created check_eligibility() function with configurable criteria:

        Income threshold: $32,275 max (based on household size)

        Supported issues: eviction, divorce, custody, benefits

        Geographic filtering by county

    Priority level logic based on issue type and urgency

    Clean CLI output with formatted results

CLI Example:

bash
python src/legal_triage.py --screen --name "Jane Doe" --income 25000 --household 3 --issue "eviction" --county "Cook County"

2. PDF Document Indexing

What it does:

    Ingests PDF legal documents into the system

    Converts PDFs to text using PyPDF2

    Creates searchable text chunks for semantic search

How it was implemented:

    Used PyPDF2 library for PDF parsing

    Text extraction with PyPDF2.PdfReader()

    Chunking strategy for manageable document segments

    Stored in data/indexed_docs.txt for search

CLI Example:

bash
python src/legal_triage.py  # Runs indexing when no arguments provided

3. Semantic Search with RAG

What it does:

    Performs semantic search over indexed legal documents

    Returns relevant document sections matching search queries

    Uses LLM for understanding search intent and context

How it was implemented:

    ** retriever pattern**: search_legal_docs(query) function

    Text chunking with overlap for better context

    LLM-based semantic matching using Perplexity API

    Formatted output with document headers, timelines, forms, and steps

CLI Example:

bash
python src/legal_triage.py "eviction process timeline"

4. Process Guides

What it does:

    Provides step-by-step legal process guides for common issues

    Shows timeline, forms needed, and important notes
    -自动生成 for eligible clients during screening

How it was implemented:

    Process guide templates stored in indexed documents

    Retrieved via semantic search matching issue type

    Structured format:

        Timeline estimates

        Required forms list

        Numbered procedural steps

        Important legal notes

Example Output:

text
=== Illinois Eviction Process ===
Timeline: 30-90 days
Forms: Notice to Quit, Complaint for Possession, Summons...
Steps:
1. Serve tenant with notice to quit (5-10 days)
2. File Complaint for Possession
3. Pay filing fee...

5. Professional CLI with Argparse

What it does:

    Clean command-line interface with proper argument handling

    Supports multiple modes: screening, search, indexing

    Error handling and help messages

How it was implemented:

    Full argparse implementation:

    python
    parser.add_argument('--screen', action='store_true')
    parser.add_argument('--name', type=str)
    parser.add_argument('--income', type=int)
    parser.add_argument('--household', type=int)
    parser.add_argument('--issue', type=str)
    parser.add_argument('--county', type=str)
    parser.add_argument('--search', type=str)
    parser.add_argument('query', nargs='?', type=str)

    Mode separation logic:

        --screen flag triggers eligibility screening

        No flag = regular semantic search

        No arguments = document indexing

    Proper validation for required screening arguments

CLI Modes:

bash
# Basic search
python src/legal_triage.py "eviction"

# Screening only
python src/legal_triage.py --screen --name "Jane" --income 25000 --household 3 --issue "eviction" --county "Cook County"

# Screening + search
python src/legal_triage.py --screen --name "Jane" --income 25000 --household 3 --issue "eviction" --county "Cook County" --search "eviction process"

# Indexing
python src/legal_triage.py

6. Priority Level System

What it does:

    Assigns priority levels to eligible clients

    1 = Critical (evictions, urgent benefits)

    2 = High (divorce, custody)

    3 = Normal (other issues)

How it was implemented:

    Priority mapping in check_eligibility():

    python
    priority_map = {
        'eviction': (1, 'Critical'),
        'benefits': (1, 'Critical'),
        'divorce': (2, 'High'),
        'custody': (2, 'High'),
    }

    Displayed in screening results

    Helps legal aid organizations triage case load

7. Referral System for Ineligible Clients

What it does:

    Shows referral information when clients don't meet eligibility

    Breaks down which criteria failed

    Suggests alternative legal services

How it was implemented:

    Criteria breakdown in check_eligibility():

        Income eligibility check with max threshold

        Issue eligibility check (supported issues list)

        Geographic eligibility check (county filtering)

    Referral message template for ineligible clients

8. RAG Architecture

What it does:

    Combines retrieval (search) with generation (LLM)

    Retrieves relevant document chunks

    LLM generates coherent, contextual responses

How it was implemented:

    Retrieval phase: Text chunking + semantic search

    Generation phase: Perplexity LLM API for response generation

    Document storage in data/indexed_docs.txt

    Search function: search_legal_docs(query)

Key Architecture Decisions

    Modular functions: Separate functions for screening, search, and inedexing

    CLI-first: Command-line interface for easy integration

    Local storage: Text files for documents (no database dependency)

    RAG pattern: Retrieval + Generation for accurate legal responses

    Priority-based triage: Helps organizations manage case load effectively
